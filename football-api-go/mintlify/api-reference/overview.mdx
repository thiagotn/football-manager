The rachao.app API v2 is a REST API. All endpoints live under `/api/v2`.

## Base URL

| Environment | URL                            |
| ----------- | ------------------------------ |
| Local dev   | `http://localhost:8080/api/v2` |
| Production  | `https://api.rachao.app/api/v2`|

## Authentication

Most endpoints require a JWT access token sent in the `Authorization` header:

```bash
curl https://api.rachao.app/api/v2/players/me \
  -H "Authorization: Bearer <ACCESS_TOKEN>"
```

Obtain a token via `POST /auth/login` or the OTP flow (`/auth/send-otp` →
`/auth/verify-otp` → `/auth/register` for new users). See the [Authentication](/authentication) guide.

## Response shape

Successful responses return the resource as JSON with a status code in the
`2xx` range. Errors return:

```json
{ "detail": "human-readable message" }
```

with a status code in the `4xx` / `5xx` range.

## Common status codes

| Code  | Meaning                                            |
| ----- | -------------------------------------------------- |
| `200` | OK                                                 |
| `201` | Resource created                                   |
| `204` | No content (successful, no body)                   |
| `401` | Missing or invalid token                           |
| `403` | Authenticated but not authorized                   |
| `404` | Resource not found                                 |
| `409` | Conflict (e.g. unique constraint, already member)  |
| `422` | Validation failure                                 |
| `500` | Internal server error                              |

## Pagination

List endpoints accept `limit` and `offset` query parameters.
